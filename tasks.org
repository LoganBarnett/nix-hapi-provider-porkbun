* Overview

This project implements a [[https://porkbun.com][Porkbun]] DNS provider for [[https://github.com/LoganBarnett/nix-hapi][nix-hapi]] — a declarative,
stateless reconciler for external service APIs.

The provider runs as a JSON-RPC subprocess (same pattern as
=nix-hapi-fake=) and implements the =Provider= trait from =nix-hapi-lib=.
It reconciles DNS records declared as Nix expressions against the live
state returned by the Porkbun API.

Reference implementations:
- Provider trait: =nix-hapi/crates/lib/src/provider.rs=
- Fake provider: =nix-hapi/crates/fake/src/lib.rs=
- Integration tests: =nix-hapi/crates/fake/tests/=

** TODO Implement plan

Implement =PorkbunProvider::plan= to diff desired state against live:

- Walk desired records; emit =Add= for records absent from live.
- Walk live records; emit =Destroy= for records absent from desired and
  not matching any =__nixhapi.ignore= pattern.
- For records present in both, compare fields and emit =Modify= if they
  differ.
- Respect =mkManaged= / =mkInitial= / =mkUnmanaged= markers from
  =nix-hapi-lib=.

** TODO Implement apply

Implement =PorkbunProvider::apply= to execute a =ProviderPlan=:

- Iterate the plan's change list.
- For =Add= → call create endpoint.
- For =Modify= → call edit endpoint with the record's Porkbun ID.
- For =Destroy= → call delete endpoint.
- Collect errors and surface them as =ProviderError= variants.

** TODO Write integration tests

Integration tests should mirror the style used in
=nix-hapi/crates/fake/tests/= but exercise Porkbun-specific behaviour.
Tests should use a test-double or recorded HTTP responses (e.g. =wiremock=
or hand-rolled mock server) rather than hitting the live API.

Suggested test cases:

- =creates_missing_record= — desired has a record, live is empty; plan
  contains one =Add=; apply calls create endpoint.
- =deletes_orphan_record= — live has a record not in desired; plan
  contains one =Destroy=; apply calls delete endpoint.
- =modifies_changed_record= — desired and live have same key but differing
  TTL or content; plan contains one =Modify=; apply calls edit endpoint.
- =no_op_when_already_converged= — desired and live match exactly; plan is
  empty; apply makes no HTTP calls.
- =ignores_exempt_records= — records matching =__nixhapi.ignore= patterns
  are not in the destroy list.
- =mkunmanaged_fields_skipped= — fields marked =mkUnmanaged= are not
  compared and do not generate =Modify= changes.
- =mkinitial_fields_not_overwritten= — fields marked =mkInitial= that
  already exist in live are not modified.
- =sensitive_fields_scrubbed_from_plan_output= — plan output does not
  contain raw =api_key= or =secret_api_key= values.
